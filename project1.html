UCEA Website Redesign

# UX Research

# UX Design

# Website Development

As a UX designer, I collaborated with the University Council for Educational Administration (UCEA) to redesign their website. Previously, UCEA engaged MemberLeap to revamp the site, but significant issues with the website's infrastructure and content organization persisted. The project involved rebuilding the information architecture and designing an improved user experience, including layout and navigation, to create a more intuitive and user-friendly website. This initiative required leveraging UX design expertise to address complex challenges and deliver a solution that met the client's needs. This project stands as a testament to delivering impactful UX design solutions.

Define Problems

Reduce Pages

The UCEA website will undergo a redesign to reduce the number of pages from 1000 to 60, focusing on eliminating outdated or redundant content. My role is to collaborate with the UCEA team to analyze, reorganize, and streamline the navigation for improved user experience. This approach will prioritize important content, making it more accessible and engaging for users.

Construct IA

The UCEA website has a complex and outdated IA due to platform shifts over time. My role is to redesign the IA with a user-centered approach, aligning it with the organizationâ€™s goals and optimizing for search engines. This will enhance navigation, improve user experience, and increase engagement through testing, feedback, and collaboration with the UCEA team.

Prototype

MemberLeap (the network provider) only provides basic migration services, all interfaces need to be designed by the client. My task is to create high-end prototypes that meet UCEA users' needs, ensuring smooth development in the next stages and enhancing user satisfaction.

Develop Website

Since MemberLeap only provides a grid and text editor-based design interface, UCEA HQ wants the website to have more interactive design elements. Additionally, considering future involvement of non-coders in website management, my responsibility is to embed web syntax (including HTML, CSS, and JavaScript) that is easy to manage, simple to understand, yet capable of supporting interactive design.

Gather Data

Analyze Stakeholders

Stakeholder analysis and interviews were conducted with UCEA members and key staff. Key stakeholder groups were identified, each with distinct roles, interests, and website requirements. This analysis provided a comprehensive understanding of the relationships between various roles and the website.

Website Audit

A comprehensive website audit, using Google Analytics to analyze traffic patterns on the UCEA website, revealed that UCEA members, the intended audience, were primarily relying on email for information rather than using the website. This insight underscores the need to improve website accessibility and usability to enhance member engagement and reduce the workload on headquarters staff.

Conduct Focus Group

A focus group of five UCEA members identified key issues with the old website, including poor information architecture, untraceable actions, inaccessible information, and redundant content. These insights are crucial for guiding the redesign, improving user-friendliness, and encouraging more members to use the site for information and resources.

Construct IA

Analyze Pages

Contextual design theory was applied to analyze the UCEA website's information architecture, creating tailored environments for different user groups. The proposed IA was visualized using Miro, a collaborative online platform, through user flow diagrams to better understand interactions and identify improvements. Miro's features facilitated effective communication with the project team and stakeholders. These user flow diagrams became crucial tools, guiding decisions to ensure the redesigned IA meets the diverse needs of UCEA website users.

Construct New IA

A tree testing tool was integrated to support the creation of a new IA for the UCEA website, based on the user flow analysis results from Miro. By presenting users with a simplified tree diagram of the site structure, this approach helps structure the IA without visual design influences. It provides valuable insights into how effectively the IA facilitates content access during the development phase.

Prototype

Lo-fi Prototype

To collect early user feedback, a lo-fi cardboard prototype of the UCEA website was created, based on the IA. User testing provided valuable insights into structure, navigation flow, and content placement, guiding necessary improvements and adjustments.

Hi-fi Prototype

Once the basic layout was finalized, an HTML and CSS testing website was developed to collect feedback on the user experience from a broader audience. Following several iterations and tests to ensure the IA and website navigation were accessible, the foundational color scheme and layout were incorporated to support further development. Development

Visual Design

In response to the client's requirements, various visual effects and layout options were iteratively developed to refine the design. CSS and JavaScript were integrated to implement micro animations, adding subtle yet impactful motion elements that enhance user interaction. Contents were carefully designed to create a more dynamic and engaging experience, making the interface feel lively without overwhelming the overall design.

Dynamic Design

Despite the basic features provided by MemberLeap, efforts were made to maintain its backend while inserting custom code to enhance aesthetics and usability. Based on the established IA, carefully designed sub-sidebars were implemented. Additionally, a shared component library was created to ensure consistent updates across the design when changes are made, streamlining the development process and improving overall site cohesion.

Revamp Content

The complex content was simplified and updated by developing detailed flowcharts that map out the information structure. Repetitive content was carefully reviewed and consolidated into unified sections, streamlining the layout and improving overall clarity and accessibility. This process ensured that all redundant elements were merged efficiently, enhancing the user experience and making the content easier to navigate.

Future Implement

A small backend system is being developed for UCEA member institutions, featuring a frontend search function. Static member institution data from an Excel file is being converted into a Django database, with a CRUD interface designed for easy management of the data. This system ensures streamlined administration and improved accessibility for users searching for member institution information.
